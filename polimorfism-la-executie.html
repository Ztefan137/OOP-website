Polimorfism la execuție (Dinamic)

Polimorfismul la execuție, cunoscut și sub numele de polimorfism dinamic sau late binding, este unul dintre pilonii fundamentali ai Programării Orientate pe Obiecte. Acesta permite unui program să determine ce metodă să execute în momentul rulării, în funcție de tipul real al obiectului.

Mecanisme fundamentale în C++

Pentru a realiza polimorfismul la execuție, avem nevoie de trei elemente esențiale:

1. Moștenire O clasă derivată trebuie să extindă în mod public o clasă de bază.
2. Funcții Virtuale Metodele din clasa de bază trebuie declarate cu sintaxa virtual.
3. Pointeri / Referințe Apelul se face prin pointeri sau referințe de tipul clasei de bază.

Exemplu de cod în C++

#include <iostream>
#include <vector>

class Animal {
public:
    virtual void scoateSunet() const {
        std::cout << "Animalul scoate un sunet generic.\n";
    }
    virtual ~Animal() = default;
};

class Caine : public Animal {
public:
    void scoateSunet() const override {
        std::cout << "Câinele latră: Ham! Ham!\n";
    }
};

class Pisica : public Animal {
public:
    void scoateSunet() const override {
        std::cout << "Pisica miorlăie: Miau! Miau!\n";
    }
};

int main() {
    std::vector<Animal*> animale = { new Caine(), new Pisica() };

    for (const auto* animal : animale) {
        animal->scoateSunet(); // Decis la execuție (dinamic)
    }

    for (auto* animal : animale) delete animal;
    return 0;
}

Ce se întâmplă în spate?

  • VTABLE (Tabela Virtuală): O tabelă statică generată de compilator ce conține adresele funcțiilor virtuale pentru fiecare clasă.
  • VPTR (Pointerul Virtual): Un pointer ascuns în interiorul fiecărui obiect care indică spre VTABLE-ul corespunzător.

Avantajele majore

  • Extensibilitate: Adaugi clase noi fără a modifica codul existent.
  • Interfețe curate: Structură uniformă pentru obiecte diferite.
  • Fără Switch-uri: Elimină structurile repetitive de verificare a tipului.
