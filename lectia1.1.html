Lecția 1.1: Introducere în POO

Descoperă paradigma care stă la baza dezvoltării software moderne: trecerea de la gândirea procedurală la modelarea prin clase și obiecte.

1. Procedural vs. Orientat pe Obiecte

Înțelegerea diferenței dintre cele două paradigme majore este esențială înainte de a scrie prima linie de cod:

Programare Procedurală

Se concentrează pe funcții și logica pașilor sequențiali. Datele sunt complet separate de procedurile care le procesează, transformând proiectele mari în sisteme greu de urmărit și de depanat.

Programare Obiectuală (POO)

Datele și comportamentele asociate sunt grupate strâns în obiecte. Această metodă oferă o mapare directă a elementelor abstracte sau fizice din lumea reală.

2. Cei Patru Piloni Fundamentali

Orice limbaj care se consideră orientat pe obiecte se bazează pe următoarele reguli arhitecturale:

01

Abstractizare

Ascunderea detaliilor de implementare complexe și evidențierea doar a trăsăturilor esențiale.

02

Încapsulare

Restricționarea accesului direct la starea internă a obiectului și oferirea de interfețe sigure de interacțiune.

03

Moștenire

Crearea de clase noi pornind de la clase existente, preluând automat atributele și metodele lor.

04

Polimorfism

Capacitatea unei singure entități (metodă sau obiect) de a se comporta diferit în contexte variate.

3. Anatomia unui Obiect: Atribute vs. Metode

Gândește-te la o Clasă ca la o schiță arhitecturală (blueprint). Ea nu există fizic, este doar un plan. Obiectul este casa construită efectiv pe baza acelui plan (instanța).

Atribute (Starea) Variabilele declarate în clasă. Ele definesc ce proprietăți are obiectul. (Exemplu: culoare, nume, pret)
Metode (Comportamentul) Funcțiile declarate în interiorul clasei. Ele definesc acțiunile pe care obiectul le poate efectua. (Exemplu: accelereaza(), afiseaza())

4. Niveluri de Vizibilitate

Pentru a implementa conceptul de încapsulare, controlăm accesul la membrii unei clase folosind etichete de vizibilitate:

Modificator Vizibil în aceeași Clasă Vizibil în Subclase (Moștenire) Vizibil din Exterior
public ✓ Da ✓ Da ✓ Da
private ✓ Da ✗ Nu ✗ Nu
protected ✓ Da ✓ Da ✗ Nu

5. Sintaxă și Exemplu Practic (C++)

#include <iostream> using namespace std; class Student { private: string nume; // Date ascunse din exterior int nota; public: void setDate(string n, int nt) { // Interfață de acces controlat nume = n; nota = nt; } void afiseazaDate() { cout << "Student: " << nume << " | Nota: " << nota << endl; } }; int main() { Student s1; s1.setDate("Alex", 10); s1.afiseazaDate(); // Încapsularea protejează starea internă! return 0; }

6. De ce folosim POO?

🧩 Modularitate avansată

Codul este structurat nativ pe module complet independente, mult mai ușor de testat și întreținut.

🔄 Reutilizare ridicată

Clasele odată scrise pot fi refolosite în siguranță în cu totul alte componente fără a risca coruperea datelor.

🔒 Securitatea datelor

Încapsularea acționează ca un scut, blocând complet modificările accidentale sosite din afara obiectului.

🌿 Extensibilitate simplă

Adăugarea de noi funcționalități mari într-un proiect masiv se face elegant, prin mecanismul de moștenire.
